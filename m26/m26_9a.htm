DATA Step Processing of Repeating Blocks of Data

This animation shows the processing of repeating blocks of data in the raw data file Tempdata.

Raw Data File Tempdata

1---+----10---+----20---+----30--
01APR90 68 02APR90 67 03APR90 70
04APR90 74 05APR90 72 06APR90 73
07APR90 71 08APR90 75 09APR90 76

  1. The following program is submitted to SAS: 
    data perm.april90; 
   infile tempdata;
   input Date : date. HighTemp @@;
   format date date9.; 
run;
    As the execution phase begins, the input pointer rests on column 1 of record 1.

  2. During the first iteration of the DATA step, the first block of values for Date and HighTemp are read and stored in the program data vector.

  3. The first observation is written to the data set, control returns to the top of the DATA step, and values are reset to missing.

  4. As the second iteration begins, _N_ is incremented to 2. The @@ prevents the input pointer from moving down to column 1 of the next record.

  5. The INPUT statement reads the second block of values for Date and HighTemp in the first record.

  6. The second observation is written to the data set, and control returns to the top of the DATA step.

  7. During the third iteration, the last block of values is read and written to the data set as the third observation. Control returns to the top of the DATA step, and values are reset to missing.

  8. During the fourth iteration, the first block of values in the second record is read and written as the fourth observation. Control returns to the top of the DATA step, and values are reset to missing.

  9. The execution phase continues until the last block of data is read.
