Sizing Graphs for Export

When you use the HSIZE= and VSIZE= options, the display area and the graphics output area might not be the same size. When you simply view the graph, this is not a problem, but the difference might become apparent when you export the graph.

For example, depending on your display, a graphic might look like the one below when it is exported. Notice that the graphics output area defined by HSIZE= and VSIZE= is not necessarily the entire display area. (The display area has been grayed to distinguish it from the graphics output area.)




To export a graph that is sized exactly as specified by HSIZE= and VSIZE=, follow these steps:
  1. Create the graph in the GRAPH window. It's useful to add a border so that you can see the exact graphics output area.

  2. Size the GRAPH window to the approximate size of the graphic. (The entire graphic might not be visible as you resize the window.)

  3. Re-submit the SAS/GRAPH program.

  4. Adjust the size of the GRAPH window until no display area appears between the border of the graph and the window border.

  5. Issue the WSAVE command to save the dimensions of the GRAPH window.

  6. Re-submit the SAS/GRAPH program to create the graph in the newly sized GRAPH window.

  7. Export the graph.


This process involves several steps, but it is useful when you want to export more than one graph of the same size interactively.
