Changes to the PRESERVE_COL_NAMES= Option

In SAS/ACCESS Release 8.2, the PRESERVE_COL_NAMES= option applies only when you use SAS/ACCESS software to create a new DBMS table. When you create a table, you assign the column names by using one of the following methods:

  • To control the case of the DBMS column names, specify variables using the desired case and set PRESERVE_COL_NAMES=YES. If you use special symbols or blanks, you must set VALIDVARNAME= to ANY and use SAS name literals. For more information, see the SAS/ACCESS documentation.

  • To allow the DBMS to normalize the column names according to its naming conventions, specify variables using any case and set PRESERVE_COLUMN_NAMES= NO.
When you use SAS/ACCESS software to read from, insert rows into, or modify data in an existing DBMS table, SAS identifies the database column names by their spelling. Therefore, when the database column exists, the case of the variable does not matter. Remember that in order to use column names that are not valid SAS names in your SAS program, you must use either the DQUOTE=ANSI option or the VALIDVARNAME=ANY option.
